# ДЗ Pro — обновление FAISS-индекса при изменении базы знаний

## Что нужно сделать

Доработать механизм работы с базой знаний так, чтобы векторный индекс FAISS автоматически пересоздавался, если исходный файл `knowledge_base.txt` был изменён. Это гарантирует, что ассистент всегда отвечает на основе актуальной информации без ручного вмешательства.

---

## Подсказка: алгоритм решения

1. **Научиться сравнивать версии файла**  
   Нужен способ понять, изменился ли `knowledge_base.txt` с прошлого раза. Проще всего использовать хеш-сумму (например, MD5) — если хеш изменился, значит файл другой.

2. **Где хранить информацию о версии**  
   Хеш нужно где-то сохранять между запусками приложения. Подойдёт отдельный JSON-файл рядом с индексом (например, `kb_hash.json`).

3. **Когда проверять актуальность**  
   Проверку нужно делать при каждом обращении к индексу — в функции `get_db_index()`. Если хеш не совпадает или индекс отсутствует — пересоздаём.

4. **Как пересоздать индекс**  
   Использовать уже готовую функцию `build_and_save_faiss_index()`, только добавить в неё сохранение нового хеша после создания индекса.

5. **Что должно получиться в итоге**  
   - При первом запуске создаётся индекс и файл с хешем
   - При последующих запусках индекс просто загружается (если файл не менялся)
   - После изменения `knowledge_base.txt` индекс автоматически пересоздаётся при следующем запросе
   - Пользователь видит в консоли сообщение о том, что индекс перестраивается

---

## Критерии готового решения

- Индекс пересоздаётся автоматически при любом изменении `knowledge_base.txt`  
- При отсутствии изменений индекс загружается из кэша (не пересоздаётся)  
- При первом запуске (нет индекса) он создаётся автоматически  
- В папке `kb/` появляется файл `kb_hash.json` с хешем последней версии базы знаний  
- В консоль выводится информационное сообщение о пересоздании индекса  

---

## Формат сдачи

1. Подготовьте архив проекта со всеми внесёнными изменениями
2. Загрузите архив на любой файловый хостинг:
   - Яндекс.Диск
   - Google Drive
   - GitHub (репозиторий)
3. Убедитесь, что доступ открыт (для Яндекс.Диска и Google Drive включите общий доступ по ссылке)
4. Пришлите ссылку на архив или репозиторий для проверки
